# User Flows

## Customer finds a truck
1. Open app/site
2. Allow location or enter city
3. View nearby trucks on map/list
4. Filter by open now / cuisine / distance
5. Tap truck
6. Review live status, menu, and location
7. Tap directions or follow truck

## Owner creates truck and goes live
1. Sign up as owner
2. Create or claim truck
3. Add basic truck info
4. Add menu basics
5. Add first stop or use current location
6. Tap Go Live
7. Publish public status

## Owner updates live state mid-service
1. Open dashboard
2. Tap current live card
3. Change status to busy / sold out / closed early
4. Save update
5. Public status reflects new state

## Owner adds future stop
1. Open schedule manager
2. Add stop
3. Enter venue, time, and address
4. Save as public stop
5. Optionally copy as template

## Host requests truck
1. Open booking request form
2. Enter contact info
3. Enter event date, venue, and notes
4. Submit request
5. Owner receives request
6. Owner accepts or declines

## Customer follows truck
1. Open truck detail page
2. Tap follow
3. Truck appears in favorites
4. Later alerts can be enabled

## Admin reviews claim
1. Open claims queue
2. Inspect claim and truck details
3. Approve or reject
4. Owner account gains truck control if approved
